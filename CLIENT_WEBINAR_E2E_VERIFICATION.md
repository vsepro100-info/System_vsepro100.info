# Минимальная end-to-end проверка клиентского вебинара (без UI)

## Цель
Задать минимальный, архитектурно безопасный способ проверить полный вебинарный пайплайн без UI, без новых плагинов и без побочных эффектов.

## Минимальная цепочка проверки (4 шага)
1) **Raw-сигнал (источник + hook):** источник `client-webinar-tracker-v2`; допустимый hook `do_action("client_webinar_entered")` как факт входа клиента.
2) **Каноническое событие:** `webinar_entered` должно быть зафиксировано как канонический факт входа.
3) **Сценарное действие:** Scenario Service должен эмитить действие «маршрутизация клиента между допустимыми состояниями сценария» как реакцию на `webinar_entered`.
4) **Обновление состояния потребителя:** Consumer маршрута клиента должен обновить состояние маршрута (зафиксировать новое допустимое состояние/шаг клиента).

## Ограничения
- Только фактологическая проверка цепочки: raw-сигнал → каноническое событие → сценарное действие → состояние потребителя.
- Никаких UI, тест‑харнесов, моков, новых hook’ов или реализации.
- Никаких побочных эффектов (уведомлений, расписаний, партнёрской логики, продаж).
