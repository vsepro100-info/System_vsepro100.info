# Зачем нужен Chat Handoff
Обеспечить непрерывность контекста между чатами и исключить потерю решений.

# Когда допустим переход в новый чат
Только после выполнения всех обязательных действий перед переходом.

# Обязательные действия ПЕРЕД переходом в новый чат
- обновлён SYSTEM_STATE.md;
- добавлена запись в SESSION_LOG.md;
- зафиксированы решения (если были).

# Обязательные действия В НАЧАЛЕ нового чата
- ссылка на SYSTEM_STATE.md;
- указание текущего этапа;
- указание одного следующего действия.

# Правило
Если Chat Handoff не выполнен — STOP.
