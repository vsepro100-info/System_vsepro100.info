# Назначение репозитория как памяти проекта
Репозиторий — единственная постоянная память проекта; только зафиксированное в нём считается существующим.

# 4 обязательных опорных документа
- **SYSTEM_STATE.md** — текущее состояние системы.
- **00_INDEX.md** — навигация по знаниям.
- **DECISIONS_LOG.md** — журнал решений.
- **SESSION_LOG.md** — краткие итоги рабочих сессий.

# Что хранится в каждом из этих файлов
- **SYSTEM_STATE.md**: активный этап, ключевые статусы, актуальные ограничения.
- **00_INDEX.md**: список и ссылки на все действующие документы.
- **DECISIONS_LOG.md**: принятые решения с датами и контекстом.
- **SESSION_LOG.md**: итоговые заметки по сессиям и передаче контекста.

# Правило
Новый документ считается существующим только если он добавлен в **00_INDEX.md**.
