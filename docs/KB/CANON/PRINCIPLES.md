# PRINCIPLES — архитектурные инварианты

- Система модульная: каждый модуль независим и имеет собственные границы.
- Взаимодействие между модулями допускается только через hooks и утверждённые контракты.
- Core владеет каноническими сущностями и событиями.
- Канонический источник истины должен быть единственным, без дублирующих версий.
- Контракты и события Core определяют совместимость модулей.
- Архитектурные решения фиксируют границы и роли, а не тактические детали.
- Конфликтующие политики и альтернативные «истины» запрещены.
