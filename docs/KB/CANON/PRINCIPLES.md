# CANON — фундаментальные инварианты

## Источник истины
- Единственный канон проекта — Knowledge Base (KB).
- Если факта нет в KB — его не существует в системе.

## Приоритеты
- CANON имеет высший приоритет над любыми другими документами.
- При конфликте следовать CANON, остальное подлежит уточнению и обновлению.

## Изменения только через PR
- Любые изменения в KB фиксируются через Pull Request.
- Изменения без PR считаются недействительными.

## Иерархия типов документов
1) CANON — фундаментальные инварианты и принципы системы.
2) SPEC — спецификации продукта и модулей.
3) PROCESS / MODE — операционные регламенты и режимы работы.
4) DRAFT — идеи и гипотезы без утверждения.
5) ARCHIVE — исторические материалы без силы нормы.
